# Руководство по развертыванию AmneziaWG-ProxyChain

В этом документе пошагово описан процесс ручной сборки цепочки прокси-серверов из двух узлов на базе протокола AmneziaWG с использованием Docker.

## Принцип работы цепочки
- **Входной узел (RU):** Принимает подключения от клиентов. Маршрутизирует трафик к заранее известным подсетям (например, российским IP-адресам) **напрямую** в интернет (SNAT). Весь остальной трафик маркируется (fwmark) и отправляется во второй туннель.
- **Выходной узел (AM/NL/др.):** Принимает туннелированный трафик от первого узла и выпускает его в глобальный интернет.

---

## Подготовка (Генерация ключей)

> **Внимание:** Все ключи в примерах ниже взяты для наглядности. Вам необходимо сгенерировать свои ключи! Сгенерировать их можно с помощью утилиты `awg genkey` и `awg pubkey`.

```bash
# Пример генерации ключей:
PRIVATE_KEY=$(awg genkey)
PUBLIC_KEY=$(echo $PRIVATE_KEY | awg pubkey)
```

Вам потребуются:
1. `RU_PUB_KEY` — публичный ключ для Входного узла.
2. `AM_PUB_KEY` — публичный ключ для Выходного узла.
3. `CLIENT_PUB_KEY` (до 5 штук) — ключи для ваших конечных устройств.

---

## Шаг 1. Развертывание Выходного узла (Сервер №2)

1. Установите Docker на удаленном сервере (например, в Нидерландах или Армении).
2. Скопируйте исходный код репозитория на сервер в папку `/root/AmneziaWG-ProxyChain`.
3. Перейдите в папку и запустите сборку и старт контейнера `server-am`, передав ему публичный ключ Входного узла (`RU_PUB_KEY`):
   
```bash
cd /root/AmneziaWG-ProxyChain
RU_PUB_KEY='<публичный_ключ_сервера_рф>' docker compose up -d --build server-am
```

4. После старта извлеките **Публичный ключ** и **IP-адрес** этого сервера, они понадобятся на следующем шаге:
```bash
docker exec awg-am cat /etc/amnezia/amneziawg/server_public_key
```

---

## Шаг 2. Развертывание Входного узла (Сервер №1 - РФ)

1. Установите Docker на вашем российском сервере.
2. Скопируйте исходный код репозитория на сервер в папку `/root/AmneziaWG-ProxyChain`.
3. Передайте собранные ключи в переменные окружения и запустите контейнер `server-ru`:

```bash
# Переменные:
# AM_PUB_KEY — публичный ключ Выходного узла (Шаг 1.4)
# AM_ENDPOINT — IP-адрес Выходного узла (с портом 51821, например 167.172.168.91:51821)
# CLIENT_PUB_KEY — публичный ключ вашего устройства

cd /root/AmneziaWG-ProxyChain

CLIENT_PUB_KEY='<ваш_клиент_pub_key>' \
AM_PUB_KEY='<ключ_узла_2>' \
AM_ENDPOINT='<ip_узла_2>:51821' \
docker compose up -d --build server-ru
```

4. Дождитесь запуска. Сервер РФ скачает список российских подсетей и настроит правила маршрутизации.
5. Извлеките **Публичный ключ Московского сервера**:
```bash
docker exec awg-ru cat /etc/amnezia/amneziawg/server_public_key
```

---

## Шаг 3. Настройка клиента

Создайте на устройстве файл конфигурации (например `client.conf`) и импортируйте его в AmneziaWG:

```ini
[Interface]
PrivateKey = <приватный_ключ_вашего_устройства>
Address = 10.8.0.2/32
DNS = 8.8.8.8, 1.1.1.1

# ⚠️ Обязательные параметры маскировки AmneziaWG, должны совпадать с сервером
Jc = 4
Jmin = 50
Jmax = 1000
S1 = 80
S2 = 120
H1 = 1
H2 = 2
H3 = 3
H4 = 4

[Peer]
PublicKey = <Публичный ключ Московского сервера из Шага 2.5>
Endpoint = <IP-адрес Московского сервера>:51820
AllowedIPs = 0.0.0.0/0
PersistentKeepalive = 25
```

## Добавление дополнительных клиентов

Сервер `server-ru` поддерживает **до 5 одновременных подключений**. 
Чтобы добавить нового клиента (без изменения кода):

1. При запуске `docker compose` на сервере РФ вы можете передать дополнительные ключи:
   - `CLIENT2_PUB_KEY`
   - `CLIENT3_PUB_KEY`
   - `CLIENT4_PUB_KEY`
   - `CLIENT5_PUB_KEY`
   
**Пример добавления второго и третьего клиента в запущенную цепочку:**
```bash
CLIENT_PUB_KEY='<ключ1>' \
CLIENT2_PUB_KEY='<ключ2>' \
CLIENT3_PUB_KEY='<ключ3>' \
AM_PUB_KEY='<ключ_узла_2>' \
AM_ENDPOINT='<ip_узла_2>:51821' \
docker compose up -d server-ru
```

В клиентских конфигах для дополнительных устройств указывайте следующие IP-адреса:
- Клиент 2: `Address = 10.8.0.3/32`
- Клиент 3: `Address = 10.8.0.4/32`
и так далее.

### Если нужно больше 5 клиентов:
1. Отредактируйте файл `server-ru/awg0.conf.template` и добавьте блоки `[Peer]` для слота `CLIENT6_PUB_KEY` и т.д.
2. Пропишите эти же переменные в `docker-compose.yml` в секции среды сервиса `server-ru`.
3. Перезапустите контейнер с передачей новых ключей.
