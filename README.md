# AmneziaWG Proxy-Chain 🛡️

Продвинутое решение на базе протокола **AmneziaWG (WireGuard с защитой от DPI)** для создания устойчивой двухузловой proxy-цепочки.

Проект позволяет развернуть два узла для прозрачного и **селективного** роутинга трафика:
1. **Входной узел (РФ)** — принимает AmneziaWG-подключения от клиентов. Трафик к российским IP-адресам пускается **напрямую** (без потери скорости), весь остальной — через второй туннель.
2. **Выходной узел (Зарубежный)** — принимает туннелированный трафик и выпускает его в глобальную сеть.

## Особенности текущей развёртки

| Параметр | Входной (RU) | Выходной (AM/NL) |
|---|---|---|
| ОС | Debian 11 (Bullseye) | Debian 13 (Trixie) |
| Ядро | `5.10.x-amd64` | `6.12.x-amd64` |
| **Модуль ядра** | ✅ `amneziawg.ko` (PPA) | ✅ `amneziawg.ko` (DKMS) |
| **Режим работы** | **Kernel-side** | **Kernel-side** |

> **Важно:** Оба сервера работают в режиме **Kernel-side** (через модуль ядра `amneziawg`), а не через userspace-реализацию `amneziawg-go`. Это обеспечивает максимальную производительность и стабильность.

## 📡 Мониторинг канала (Packet Loss)

Скрипт `monitor_tunnel.sh` периодически отправляет ICMP-пакеты между узлами и записывает данные о потерях и задержках в `/var/log/awg_monitor.log`.

```bash
# Запустить на RU-сервере (мониторинг канала до AM-сервера):
./monitor_tunnel.sh <IP_AM_СЕРВЕРА>
```

Если потери пакетов превышают **20%**, в лог записывается предупреждение `WARNING`. На этой основе можно добавить алерты в Telegram или Slack.

## ❓ О совместимости с VLESS

AmneziaWG работает поверх **UDP** и модифицирует заголовки пакетов для обхода DPI. VLESS работает поверх **TCP/WebSocket/gRPC** (Xray-core/Sing-box) — это принципиально разные транспортные уровни, поэтому встроить один в другой технически невозможно.

**Варианты совместного использования:**
1. Поднять Xray-core (VLESS) **параллельно** на том же узле — как резервный транспорт.
2. Завернуть VLESS-трафик **через** существующий AmneziaWG-туннель.
